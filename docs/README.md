# Documentação do Projeto

Bem-vindo à pasta `docs/` do projeto.

## Objetivo
Organizar as tarefas e issues já concluídas em arquivos Markdown (`.md`) para que a documentação fique clara e acessível.

## Como começar
1. Abra o GitHub e veja as issues ou cards que já foram entregues.
2. Para cada funcionalidade ou tarefa importante, crie um arquivo Markdown em `docs/` com título e descrição.
3. Use este formato básico:

```md
# Título da funcionalidade

- Descrição breve do que foi feito
- Tecnologias ou componentes afetados
- Como testar ou verificar
```

## Exemplo de arquivo de documentação
Crie arquivos como `docs/issue-123.md`, `docs/planejamento.md` ou `docs/conclusoes.md`.

## Próximos passos
- Se já tiver issues mapeadas, copie o texto principal e transforme em `.md`
- Se não tiver certeza do conteúdo, crie um documento com perguntas como "O que foi feito?" e "Por que isso é importante?"
- Atualize o README do projeto depois com links para os arquivos em `docs/`
